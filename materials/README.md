# Materials Overview

This directory stores the consolidated project materials imported from the three original workstreams. It exists to keep the final repository self-contained and to preserve the experimental artifacts that informed the integrated deliverable.

## Contents

- `project_part1/`
  Data processing pipeline, chunking design, OCR comparison workflow, and corpus-processing notes.
- `project_part2/`
  Retrieval experiments, Top-K result tables, and embedding-related assets.
- `Project_part_3/`
  Generation and evaluation artifacts, rerun outputs, and result figures.

## Purpose

The materials folder is not intended to be the main runtime entry point of the final system. Instead, it serves as the preserved experimental foundation behind the integrated application in the root repository. The backend and frontend reference selected outputs from this directory to power the final demo and dashboard.
